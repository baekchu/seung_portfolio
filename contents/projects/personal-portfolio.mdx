## 프로젝트 개요

개발자의 프로젝트·기술 스택·활동 데이터를 한곳에 모아 보여주는 반응형 포트폴리오 사이트입니다. 단순한 소개 페이지에서 그치지 않고, 외부 서비스의 활동 데이터를 실시간으로 끌어와 보여주는 대시보드와 다국어·다크모드를 지원하는 게 특징입니다.

| 항목 | 내용 |
|---|---|
| 개발 형태 | 개인 프로젝트 |
| 프레임워크 | Next.js 14 (App Router) |
| 배포 환경 | Vercel |
| 서비스 주소 | [seung-portfolio.vercel.app](https://seung-portfolio.vercel.app/) |
| 저장소 | [github.com/baekchu/seung_portfolio](https://github.com/baekchu/seung_portfolio) |
| 지원 언어 | 한국어 / English |

## 핵심 기능 구성

### 홈 & 소개 (About)
프로필, 한 줄 소개, 기술 스택 마퀴(marquee), 최근 활동을 한눈에 보여주는 랜딩 영역입니다. `AOS`와 `Framer Motion`으로 스크롤·등장 애니메이션을 적용했습니다.

### 대시보드 (Dashboard)
여러 외부 서비스의 활동 데이터를 API로 수집해 카드·차트로 시각화합니다.

| 위젯 | 데이터 출처 |
|---|---|
| GitHub Contributions | GitHub 잔디(기여도) 캘린더 |
| Coding Activity | WakaTime 코딩 시간 통계 |
| Codewars | 코딩 챌린지 랭크·점수 |
| Monkeytype | 타이핑 속도(WPM) 리더보드 |
| Umami Analytics | 사이트 방문 트래픽 추이 |

`chart.js` + `react-chartjs-2`로 기여도 캘린더와 트래픽 추이 차트를 그리고, 로딩 중에는 `react-loading-skeleton` 기반 스켈레톤 UI를 노출합니다.

### 프로젝트 (Projects)
프로젝트 카드 그리드와 상세 페이지로 구성됩니다. 추천(featured) 프로젝트를 우선 정렬하며, 각 상세 페이지는 MDX 문서로 작성되어 기술 스택·트러블슈팅 등 풍부한 내용을 담습니다. (지금 보고 있는 이 페이지가 바로 그 구조입니다.)

### 연락처 (Contact)
`react-hook-form`으로 검증하는 문의 폼이며, 제출 시 `nodemailer`로 이메일이 발송됩니다.

### 다국어 · 다크모드
`next-intl`로 한/영 전체 UI를 번역하고, `next-themes`로 라이트/다크 테마를 전환합니다.

## 디렉터리 구조

기능(module)·공통(common)·서비스(service) 레이어로 관심사를 분리했습니다.

```
app/                  # App Router 페이지 & API Route
  ├── page.tsx        # 홈
  ├── about/  dashboard/  projects/  contact/
  └── api/            # github · codewars · monkeytype · umami · projects ...
modules/              # 기능 단위 컴포넌트 (dashboard, projects, contact, about ...)
common/
  ├── components/     # 재사용 UI (elements, layouts)
  ├── constants/      # 프로젝트·스택 등 정적 데이터
  ├── stores/         # Zustand 스토어 (theme, menu, notif)
  └── types/          # 타입 정의
services/             # 외부 API 호출 로직 (fetcher 포함)
contents/projects/    # 프로젝트 상세 MDX 문서
messages/             # kr.json · en.json 번역 리소스
```

## 대시보드 — 외부 API 집계

각 위젯은 클라이언트에서 `SWR`로 내부 API Route(`/api/*`)를 호출하고, 서버 측 `services/` 레이어가 외부 서비스와 통신합니다. API 키 같은 민감 정보는 서버에서만 다루고 클라이언트로는 내려보내지 않습니다.

```
[Client] SWR(useSWR) ──▶ [/api/github] ──▶ services/github ──▶ GitHub API
                          (캐싱·재검증)        (토큰은 서버 전용)
```

- `stale-while-revalidate`로 첫 진입 시 캐시를 즉시 보여주고 백그라운드에서 갱신
- 로딩 상태는 스켈레톤, 오류 시 `EmptyState`로 graceful degradation

## 다국어(i18n) 구현

`next-intl`을 사용해 `messages/kr.json`·`messages/en.json`에 모든 텍스트를 키 기반으로 관리합니다. 프로젝트 데이터처럼 동적 콘텐츠도 `ProjectsData.{id}` 네임스페이스로 번역을 매핑하여, 코드 변경 없이 메시지 파일만으로 언어를 추가·수정할 수 있습니다.

```typescript
// 서버에서 로케일별 프로젝트 데이터 변환
const t = await getTranslations({ locale, namespace: "ProjectsData" });
return PROJECTS.map((p) => ({
  ...p,
  title: t(`${p.id}.title`) || p.title,
  description: t(`${p.id}.description`) || p.description,
}));
```

## MDX 기반 프로젝트 상세

프로젝트 상세 본문은 `contents/projects/{slug}.mdx` 파일로 분리해 관리합니다. 빌드 시 MDX를 읽어 `slug`로 매칭하고, `react-markdown` + `remark-gfm`으로 렌더링하며 표·코드블록·인용구에 커스텀 스타일을 적용합니다. 콘텐츠와 코드를 분리해둔 덕분에 글만 고쳐도 상세 페이지가 갱신됩니다.

## 트러블슈팅

### 다크모드 하이드레이션 불일치
서버 렌더링 시점에는 테마를 알 수 없어 `next-themes` 적용 전 깜빡임/하이드레이션 경고가 발생했습니다. 테마 의존 UI는 마운트 이후 렌더링하도록 처리하고 `suppressHydrationWarning`을 적용해 해결했습니다.

### SWR 중복 요청과 캐싱
대시보드의 여러 위젯이 동일 데이터를 각각 호출하며 불필요한 요청이 발생했습니다. 공통 `fetcher`와 SWR 키 통일, `revalidateOnFocus` 조정으로 호출 횟수를 줄였습니다.

### 외부 API 레이트 리밋
GitHub·Umami 등 외부 API의 호출 제한에 대응하기 위해, 토큰을 서버 환경변수로 분리하고 API Route 단에서 응답을 가공·캐싱하여 클라이언트의 직접 호출을 차단했습니다.

### MDX 상세 페이지 라우팅
존재하지 않는 `slug` 접근 시 오류가 나던 문제를, 데이터 조회 실패 시 `null` 처리 및 `not-found` 페이지로 안전하게 폴백하도록 개선했습니다.

## 배포 구성

Vercel에 GitHub 연동으로 `main` 브랜치 push 시 자동 배포됩니다. 외부 API 토큰·이메일 계정 같은 민감 정보는 전부 Vercel 환경변수로 빼서 소스코드에 노출되지 않게 했습니다. `@vercel/analytics`로 방문 데이터를 수집합니다.
