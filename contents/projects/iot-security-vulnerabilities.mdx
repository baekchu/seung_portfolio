## 연구 개요

**「사물인터넷(IoT, Internet of Things)의 초연결 사회에 따른 보안 취약점과 대응 방안」**은 인터넷에 연결된 사물(디바이스)이 폭발적으로 늘어나는 초연결 사회에서 필연적으로 발생하는 보안 취약점을 분석하고, 실제 공격 시나리오와 단계별 대응 방안을 정리한 연구 보고서입니다.

## 1. 배경 — IoT 시장과 보안의 동반 성장

스마트 홈, 스마트 헬스케어, 스마트 자동차, 스마트 에너지 등 다양한 응용 환경에서 사물이 지능형 스마트 사물로 진화하면서 IoT 시장이 급속히 성장하고 있습니다.

- **글로벌 IoT 지출액**: 2023년까지 연평균 **12.6%** 성장 전망
- **IoT 플랫폼 시장**: 2018~2023년 연평균 **39%** 성장, 223억 달러 규모 전망
- **IoT 보안 시장**: 2017년 84억 달러 → 2025년 **309억 달러** 전망 (연평균 17.6%)

플랫폼 시장이 커질수록 보안 시장도 함께 성장하며, 블록체인 상용화로 IoT 보안 수요는 더욱 확대될 것으로 예측됩니다.

## 2. IoT의 10대 취약점 (OWASP IoT Top 10)

1. 쉬운 암호·유추 가능한 암호·하드코딩된 암호 사용
2. 안전하지 않은 네트워크 서비스
3. 안전하지 않은 생태계 인터페이스
4. 안전한 업데이트 메커니즘의 부재
5. 안전하지 않거나 오래된 구성요소 사용
6. 불충분한 개인정보 보호
7. 안전하지 않은 데이터 전송 및 저장
8. 디바이스 관리의 부재
9. 안전하지 않은 기본 설정
10. 물리적 보호 수단의 부재

## 3. 공격 시나리오 — CoAP 프로토콜 악용

IoT 디바이스가 웹 기반 응용 프로토콜인 **CoAP**으로 웹에 접속하는 특성을 노린 공격 흐름입니다.

```
① 해커가 이용자 많은 홈페이지를 해킹 → 악성 스크립트 삽입
② 디바이스가 CoAP 프로토콜로 해당 웹에 접속
③ 접속 순간 사전 삽입된 악성 스크립트 실행 → 디바이스 악성코드 감염
④ 악성코드를 통해 해커가 디바이스 권한 획득
```

CoAP은 웹 기반 프로토콜이기 때문에 **기존 웹 해킹 기법에 그대로 노출**된다는 점이 핵심 위협입니다.

---

## 4. 주요 공격 기법

### 펌웨어 공격
기기에 탑재된 펌웨어를 분석해 특정 포트·입력 대기 프로세스를 찾아내고, 직접 수정한 가짜 악성 펌웨어를 주입해 기기 자체를 장악합니다. 장악된 기기는 좀비 PC처럼 **DDoS 공격**이나 **암호화폐 채굴**에 동원되며, 망가진 펌웨어로 장치를 무력화하기도 합니다.

> **공급망 공격 사례** — 2018년 ASUS 라이브 업데이트 해킹: 공격자가 업데이트 서버를 장악하고 정상 SW 업데이트로 위장해 100만 명 이상의 사용자 PC에 백도어를 설치했습니다.

### 공유기 장악
IoT는 공유기를 통해 데이터를 주고받으므로, 취약점이 있거나 관리자 비밀번호가 약한 공유기를 해킹해 장악한 뒤 오가는 트래픽을 관찰하여 **IoT 게이트웨이를 식별**합니다.

### 감시카메라(CCTV) 공격
CCTV에서 발견되는 **스택 오버플로우** 취약점 — 버퍼 할당량보다 많은 데이터를 쓸 때 발생 — 을 악용해 권한 상승으로 카메라를 장악하고, 범죄용 정보 수집이나 기록 중단을 시도합니다.

### 어택 서피스(Attack Surface) 공격
권한 없는 사용자가 데이터를 입력·추출할 수 있는 입력 필드·프로토콜·인터페이스·서비스 등 **공격 표면**을 노립니다.

## 5. 대응 방안

사용자단에서 가능한 조치가 제한적이므로, **설계 단계에서의 철저한 초기 보안 설계**가 가장 중요합니다.

**개인 차원**
- 공유기 포트포워딩 비활성화
- IoT 기본(default) ID·패스워드 변경
- 디바이스 최신 보안 패치 적용 및 백신 실시간 검사

**웹 운영자 차원**
- CoAP이 웹 기반인 만큼 보안 패치·백신 등 웹 보안 조치 수행

### 빅데이터(Hadoop) 기반 국가적 대응

```
① 국가 차원의 빅데이터 플랫폼 구축
② Hadoop으로 각 웹사이트 안전성 검사
③ 위험 사이트를 블랙리스트에 등록
④ CoAP로 웹 접속하는 IoT 디바이스가 국가 제공 블랙리스트를 점검
⑤ 블랙리스트 미등록 사이트일 경우에만 정상 접속
```

빅데이터 시장에서 보편적으로 쓰이는 **Hadoop**을 활용해 비정상 웹페이지를 사전 차단하는 구조입니다.

## 6. 결론

IoT 기기는 음성 제어, 조명·커튼 자동화 등으로 삶의 질을 높이며 점점 더 사람의 삶에 밀접해지고 있습니다. 그러나 편리함 뒤에는 해킹 위험이 존재하며, 방치할 경우 개인 차원을 넘어 **국가적 테러와 인명을 위협하는 수단**으로 악용될 수 있습니다. 따라서 다가오는 IoT 시대의 위험성을 인식하고, 지속적으로 발전된 보안 대책과 대응 방안을 마련해야 합니다.
